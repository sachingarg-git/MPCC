1. Introduction & Overview

1.1 What is MPCC?

MPCC (Medical Pollution Control Company) is a comprehensive Biomedical Waste Management System designed to streamline the collection, tracking, and disposal of biomedical waste from healthcare facilities (HCFs) in the Haridwar and Rishikesh regions of Uttarakhand, India.

🎯 Purpose: To provide a digital platform for healthcare facilities to register for biomedical waste collection services, manage their subscriptions, track pickups, and maintain compliance with environmental regulations.

Target Users

  • Healthcare Facilities (HCFs): Hospitals, Clinics, Nursing Homes, Diagnostic Labs, Pathology Labs, Dental Clinics, etc.
  • MPCC Administrators: Staff managing operations, routes, and customer relationships
  • Field Staff: Drivers and collection personnel

1.2 System Architecture

MPCC is built as a modern full-stack web application with the following components:

Component Technology Description
Frontend React + Vite + TypeScript Modern, responsive user interface with Tailwind CSS styling
Backend Node.js + Express.js RESTful API server handling all business logic
Database Microsoft SQL Server (MSSQL) Relational database for data persistence
Authentication bcrypt + Session-based Secure password hashing and user authentication

1.3 Key Features

📝 Customer Registration

Step-by-step wizard for healthcare facilities to register for waste collection services with category and plan selection.

📦 Service Plan Management

Flexible service plans with customizable pricing based on facility type, bed count, and waste categories.

🗺️ Zone & Route Management

Organize service areas into zones with optimized collection routes for efficient operations.

🚛 Pickup Tracking

Track waste collections with bag counts (Yellow, Red, Green, Black), weights, and vehicle assignments.

💰 Billing & Invoicing

Automated invoice generation with multiple payment frequencies and online payment integration.

📜 Certificate Management

Issue and track compliance certificates for registered healthcare facilities.

🎫 Support Ticketing

Built-in helpdesk system for customer support requests and issue tracking.

👥 User & Role Management

Comprehensive user management with role-based access control (RBAC).

2.2 Installation Guide

Step 1: Clone the Repository

git clone https://github.com/sachingarg-git/MPCC.git
cd MPCC

Step 2: Install Backend Dependencies

cd backend
npm install

Step 3: Install Frontend Dependencies

cd ../frontend
npm install

2.3 Database Setup

Step 1: Create Database

Open SQL Server Management Studio (SSMS) and create a new database:

CREATE DATABASE MPCC_DB;

Step 2: Run Setup Scripts

Execute the following SQL scripts in order:

  1. DATABASE_MASTER_DATA.sql - Creates master data tables (Routes, Zones, Service Plans, etc.)
  2. CUSTOMER_PORTAL_SETUP.sql - Creates customer-related tables
  3. DATABASE_USER_MANAGEMENT.sql - Creates user and role tables

Step 3: Configure Environment Variables

Create a .env file in the backend folder:

# Database Configuration
DB_HOST=localhost
DB_PORT=1433
DB_NAME=MPCC_DB
DB_USER=your_username
DB_PASSWORD=your_password

# Server Configuration
PORT=5060
⚠️ Security Note: Never commit the .env file to version control. It contains sensitive database credentials.

2.4 Running the Application

Start the Backend Server

cd backend
npm start

The backend API will start on http://localhost:5060

Start the Frontend Development Server

cd frontend
npm run dev

The frontend will start on http://localhost:5173 (default Vite port)

✅ Success! Once both servers are running, open your browser and navigate to:
  • Customer Portal: http://localhost:5173
  • Admin Panel: http://localhost:5173/admin (or use admin.html)

3. Customer Portal (Landing Page)

The Customer Portal is the public-facing website where healthcare facilities can learn about MPCC services and register for biomedical waste collection.

3.1 Service Plan Selection

The registration process begins with selecting a service category and plan:

Waste Categories

Bag Color Waste Category Examples
Yellow Infectious Waste Anatomical waste, soiled waste, expired medicines
Red Contaminated Waste Contaminated recyclable waste (plastics, tubing)
Green General Waste Non-contaminated recyclables
Black Hazardous Waste Sharps (needles, blades), cytotoxic waste

Service Plans Structure

Plans are organized by:

  • Category: Yellow Bag, Red Bag, Multi-Category, etc.
  • Sub-Category: Specific waste types within each category
  • Zone: Geographic service area (Haridwar North, Haridwar South, Rishikesh Zone)
  • Pricing Type: Fixed monthly rate or per-bed pricing

3.2 Registration Wizard

The registration wizard guides HCFs through a multi-step process:

Step 1: Category Selection

Choose the primary waste category based on your facility type and waste generation.

Step 2: Facility Details

  • Institution Name
  • Institution Type (Hospital, Clinic, Lab, etc.)
  • Number of Beds (for hospitals/nursing homes)
  • BMW Registration Number (if available)
  • Full Address with Zone selection
  • GPS Coordinates (optional)

Step 3: Contact Information

  • Contact Person Name and Designation
  • Mobile Number and Email
  • Alternate Contact (optional)
  • Website (optional)

Step 4: Billing & Tax Details

  • PAN Number
  • GST Number
  • Billing Cycle Preference
  • Payment Mode Preference

Step 5: Plan & Kit Selection

  • Select Service Plan based on requirements
  • Choose Optional Kit (waste bins, bags, containers)
  • Add-on Services (Consulting, Compliance assistance)

Step 6: Review & Payment

  • Review all entered information
  • View cost breakdown
  • Complete payment

3.3 Payment Processing

MPCC supports multiple payment methods:

  • Online Payment: UPI, Debit/Credit Cards, Net Banking (via Razorpay)
  • Offline Payment: Cheque, Cash, Bank Transfer

4. Admin Panel

The Admin Panel is the control center for MPCC staff to manage all aspects of the biomedical waste management operations.

4.1 Dashboard Overview

The dashboard provides at-a-glance metrics organized into three sections:

Customer Overview

  • Total Registrations: All HCFs registered in the system
  • Approved/Active: Facilities with active service
  • Pending Review: New registrations awaiting approval
  • Total Revenue: Sum of all registration payments

Master Data Overview

  • Zones: Active/Total service zones
  • Routes: Active/Total collection routes
  • Service Plans: Active/Total service offerings
  • Kits: Active/Total equipment kits

CRM/HCF Overview

  • Categories: Facility type classifications
  • Raw Materials: Inventory items in stock
  • HCF Registered: Healthcare facilities by zone
  • This Month: New registrations in current month

4.2 User Management

Managing Users

Admin users can:

  • Create new user accounts with email and password
  • Assign roles (Admin, Manager, Operator, etc.)
  • Enable/disable user accounts
  • Set email notifications preferences

Role-Based Access Control

Each role has specific permissions defining what features and data they can access:

Role Description Access Level
Super Admin Full system access All modules
Admin Administrative operations Users, Reports, Settings
Manager Operations management CRM, Routes, Pickups
Operator Day-to-day operations Limited CRM access

4.3 Master Data Management

Zone Master

Define geographic service areas:

  • Zone Code: Unique identifier (e.g., ZNE-001)
  • Zone Name: Descriptive name (e.g., "Haridwar North")
  • Zone Type: Urban, Rural, Industrial, etc.
  • Description: Coverage details and boundaries

Route Master

Configure collection routes:

  • Route Code: Unique identifier (e.g., RTE-001)
  • Route Name: Descriptive name (e.g., "Daily Route A")
  • Route Type: Daily, City, Weekly, On-Demand
  • Primary Driver: Assigned driver
  • Secondary Driver: Backup driver

Service Plans

Create and manage service offerings:

  • Plan name and code
  • Category and sub-category assignment
  • Zone and route linkage
  • Pricing configuration (fixed or per-bed)
  • Included materials list

Kit Master

Define equipment kits for facilities:

  • Kit name and type (Hospital, Clinic, Lab)
  • Selling price and cost price
  • Item list with HSN codes, quantities, and rates

Raw Materials

Inventory management for supplies:

  • Material name and type (Polybag, Container, Bin, etc.)
  • HSN Code and Unit of Measure
  • Stock levels (min, max, reorder quantity)
  • Supplier information

Vehicle Master

Fleet management:

  • Vehicle registration and chassis details
  • Capacity and fuel type
  • Route assignment
  • Inspection and permit due dates
  • Insurance expiry tracking

Vendor Master

Supplier and service provider management:

  • Vendor details and contact information
  • GST, PAN, and registration numbers
  • Bank account details for payments
  • Credit terms and limits

View and manage all healthcare facility registrations:

Renewals Management

  • Track contract renewal dates
  • Configure auto-renewal settings
  • Send renewal reminders
  • Re-sign MoUs

Deregistration Process

Handle facility exit:

  • Receive letterhead request
  • Return certificate and agreement
  • Clear outstanding payments
  • Return equipment/kits
  • Close hologram

Support Tickets

Customer support management:

  • Create and assign tickets
  • Set priority levels
  • Track ticket status and history
  • Add resolutions and notes

4.5 Reports & Analytics

Generate reports for operational insights:

B. Troubleshooting

Database Connection Issues

  • Verify SQL Server is running
  • Check firewall allows port 1433
  • Confirm credentials in .env file
  • Ensure SQL Server allows TCP/IP connections

Frontend Not Loading

  • Verify Node.js is installed
  • Run npm install in frontend folder
  • Check for port conflicts
  • Clear browser cache

API Errors

  • Check backend console for error messages
  • Verify database tables exist
  • Test API with health check: /api/health
